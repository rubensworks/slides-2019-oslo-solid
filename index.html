Het Web is fundamenteel decentraal

  • Iedereen kan onafhankelijk pagina's publiceren.
  • Webpagina's kunnen verwijzen naar elkaar met behulp van hyperlinks.
  • Gebruikers kunnen links volgen om nieuwe pagina's te ontdekken.

Problemen decentraal oplossen: complex

Concurrentie voor centrale oplossingen

  • Visualisatie en gebruik van het Web

    1995 - 2001: Internet Explorer als Web browser (The Browser Wars)
    2004 - 2018: Opkomst van Chrome en mobiel surfen

  • Indexering en opzoeken op het Web

    Yahoo → Google

  • Sociale interactie op het Web

    MySpace → Facebook, Twitter

Gevaren van centrale oplossingen

  • Misbruik van persoonlijke informatie

    Centrale data opslag vereenvoudigt misbruik van informatie. (PRISM, GDPR)

  • Vendor-lock in

    Sterke afhankelijkheid van een bepaalde oplossing.

  • Censuur en beïnvloeding van inhoud

    Onderdrukking van ideëen niet conform met autoriteit. (filter bubble)

Solid: een decentrale oplossing

  • Volledig beheer over eigen data Misbruik van persoonlijke informatie

    Gebruikers kiezen zelf waar data opgeslaan staat, en wie toegang heeft.

  • Data en applicaties zijn onafhankelijk Vendor-lock in

    Gebruikers kunnen zonder problemen alternatieve applicaties gebruiken.

  • Betere visibiliteit van interne processen Censuur en beïnvloeding van inhoud

    Interne data uitwisseling is zichtbaar zodat onetische manipulatie gedetecteerd kan worden.

Solid ontkoppelt data van applicatie

  • Solid gebruikers hebben persoonlijke data pods.
    (persoonlijk gegevens, foto's, vrienden, ...)
  • Applicaties zijn onafhankelijk van pods, en vereisen lees/schrijf rechten.
  • Data opgeslaan volgens Linked Data principes.

Vrijheid waar jouw eigen data pod staat

  • Persoonlijke data pod op eigen server

    Iedereen kan een Solid server opzetten

  • Data opslag huren

    Gegarandeerde beschikbaarheid

  • Pod op een gedeelde server (overheidsdienst?)

Applicaties combineren data over meerdere Solid pods

  • Combineren van data pods

    Applicaties zijn visualisaties over één of meerdere pods.

  • Decentrale data opslag

    Gegevens worden niet centraal opgeslaan, maar bij de gebruikers zelf.

  • Expliciete toegangscontrole

    Applicaties kunnen data pas lezen of aanpassen na goedkeuring.

Een herbruikbaar account

  • Je eigen WebID

    Met de WebID standaard ben je identificeerbaar (en authenticeerbaar) via een unieke URL.

  • Inloggen op verschillende plaatsen

    Alle applicaties die je data willen gebruiken zullen je vragen op in te loggen met je WebID.

  • Volledig in eigen handen

    Niet afhankelijk van bedrijven als Google of Facebook.

Makkelijker data delen met organisaties

  • Beperkte toegang verlenen tot persoonlijke gegevens

    Organisaties kunnen lees of schrijf toegang krijgen tot bepaalde gegevens (email, woonplaats, rekeningnummer, ...)

  • Slechts één keer opslaan

    Een einde aan de fysieke en digitale kopieën van gegevens.

Nieuwe generatie sociale netwerken

  • Decentrale data opslag

    Berichten, reacties en likes worden opgeslaan in aparte data pods.

  • Compatibiliteit over verschillende applicaties

    Gebruikers moeten niet noodzakelijk dezelfde applicaties gebruiken om te interageren, aangezien enkel de data onafhankelijk is van de applicaties.
